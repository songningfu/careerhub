# Offerly V1 · 腾讯云 CloudBase 一键部署指南

> 目标：把 `careerhub_v1.html` 放到腾讯云上，给真实用户使用。
> 全程不需要买服务器，CloudBase 免费额度足够 100 人内的小团队使用。
> 预计耗时：**首次部署 30 分钟**，之后更新只需 30 秒。

---

## 一、注册腾讯云 CloudBase（5 分钟）

1. 打开 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)
2. 用微信/QQ 扫码登录
3. 点 **「立即开通」** → 选 **「按量计费」**（不会自动扣费，免费额度够用）
4. 创建环境：
   - **环境名称**：随便填，如 `careerhub-prod`
   - **环境类型**：选「按量计费」
   - **地域**：选**上海**或**广州**（离用户最近）
5. 创建完成后，记下你的 **环境 ID**（形如 `careerhub-prod-3g8xxxxx`）

---

## 二、配置认证方式（3 分钟）

进入你的环境 → 左侧 **「身份认证」**

### 1. 启用「邮箱登录」
- 点 **「登录方式」** → 找到 **「邮箱登录」** → 点 **「启用」**
- 勾选 ✅ 允许用户自助注册
- ✅ 自动验证邮箱（演示阶段建议关，避免要求用户去邮箱确认）

### 2. 启用「匿名登录」（关键！让访客可以体验）
- 同样在「登录方式」找 **「匿名登录」** → 点 **「启用」**

---

## 三、创建数据库集合（10 分钟）

进入 **「数据库」** → 点 **「+ 新建集合」**，依次创建以下 10 个集合：

| 集合名 | 用途 | 重要 |
|---|---|---|
| `applications` | 投递记录 | ⭐⭐⭐ |
| `jobs` | 意向岗位 | ⭐⭐⭐ |
| `resumes` | 简历库 | ⭐⭐⭐ |
| `opportunities` | 公司库（爬虫数据） | ⭐⭐ |
| `events` | 日历事件 | ⭐⭐⭐ |
| `companies` | 公司补充信息 | ⭐ |
| `interviews` | 面试记录 | ⭐ |
| `journal` | 复盘日记（V1 未启用） | ⭐ |
| `skills` | 技能（V1 未启用） | ⭐ |
| `settings` | 用户偏好 | ⭐⭐⭐ |

> 提示：每个集合都点 **「+ 新建集合」** → 输入名字 → 保存即可，不需要预设字段。

### 给每个集合配置「安全规则」（必须做！）

对**每个**集合：进入集合 → 上方 **「安全规则」** Tab → 选 **「自定义安全规则」** → 粘贴：

```json
{
  "read": "auth.openid == doc._openid",
  "write": "auth.openid == doc._openid"
}
```

> **这条规则的含义**：用户只能读写自己创建的数据，互相不能看到对方的简历/投递记录。这是隐私保护的核心。

**例外**：`opportunities` 集合（公司库）建议设置为「公开读、登录写」：

```json
{
  "read": true,
  "write": "auth.loginType != 'ANONYMOUS'"
}
```

> 这样所有用户都能看到公司库，但匿名访客不能改公开数据。

---

## 四、创建文件存储桶（2 分钟）

简历 PDF 文件存这里。

进入 **「存储」** → 默认就有一个 bucket（无需另建）→ 点 **「权限设置」** → 配置：

```json
{
  "read": "auth.openid == resource.openid",
  "write": "auth.openid == resource.openid",
  "delete": "auth.openid == resource.openid"
}
```

> 这样每个用户只能上传/下载/删除自己的简历。

---

## 五、配置静态托管 + 部署前端（5 分钟）

### 5.1 编辑 careerhub_v1.html

打开下载的 `careerhub_v1.html`，搜索：

```js
const CLOUDBASE_ENV_ID = window.CLOUDBASE_ENV_ID || 'careerhub-YOUR-ENV-ID';
```

把 `careerhub-YOUR-ENV-ID` 替换成你在 **第一步** 拿到的环境 ID。

### 5.2 配置 LLM API（可选）

如果你已经部署了上一回的 `careerhub-api`（JD 智能解析后端），同样搜索：

```js
const JD_API_ENDPOINT = window.JD_API_ENDPOINT || 'https://your-app.vercel.app/api/parse-jd';
```

改成你的真实 URL。**不改也能用**，但 JD 解析只能走本地正则（准确率约 60%）。

> 注：用户登录后可以在「设置 → 偏好 → JD 解析 API 端点」里自己改。

### 5.3 上传到 CloudBase 静态托管

进入 **「静态网站托管」** → 首次使用会让你开通 → **「上传文件」** → 选择你修改后的 `careerhub_v1.html`

> **改名建议**：把文件名改成 `index.html` 上传，这样访问根域名直接就是登录页。

上传完成后，你会得到一个域名，形如：

```
https://careerhub-prod-3g8xxxxx-1234567890.tcloudbaseapp.com
```

**用这个 URL 打开就是完整可用的 Offerly。** 把这个 URL 发给你的同学测试。

---

## 六、绑定自定义域名（可选）

如果你有自己的域名（如 `careerhub.cn`）：

1. 静态托管 → **「域名设置」** → **「添加域名」** → 输入 `careerhub.cn`
2. 按提示在域名服务商处添加 CNAME 解析
3. CloudBase 会自动签发 HTTPS 证书

---

## 七、更新部署（30 秒）

之后每次改了 `careerhub_v1.html`：

1. 静态托管 → **「文件管理」** → 找到 `index.html` → 点 **「替换文件」**
2. 浏览器强刷一下（Ctrl+Shift+R）就能看到新版

## 八、运营建议

### 监控用量
- 「监控告警」可以看到每天的 PV、读写次数、存储用量
- 免费额度：
  - 数据库读：5 万次/天
  - 数据库写：3 万次/天
  - 存储下载：5GB/天

## 九、常见问题

### Q1：用户登录后看不到自己的数据？
检查 **安全规则** 是不是配错了。规则必须写 `auth.openid == doc._openid`，不是 `_id`。

### Q2：上传简历报错「权限不足」？
存储桶的权限规则没配。回到 **第四步** 重新设置。

### Q3：JD 智能解析提示「网络调用失败」？
要么是 LLM API 没部署，要么是 `JD_API_ENDPOINT` 没改。本地正则解析仍可用，是兜底。

### Q4：匿名登录的用户换设备/清缓存后数据丢失？
正常现象。匿名登录的 openid 是浏览器级别的临时身份。引导用户升级为正式邮箱账号即可永久保留数据。

### Q5：CloudBase 收费吗？
**用量很小的话完全免费**。免费额度详见：[https://cloud.tencent.com/document/product/876/41522](https://cloud.tencent.com/document/product/876/41522)
学生项目一年花费通常不超过 50 块。
